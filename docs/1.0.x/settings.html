General Settings

Table of Contents
Introduction

The General Settings interface allows you to configure the software options as you like. There are a number of settings that you can adjust, and each of them is described in this section of the manual.

Basic Settings
    Document Root - This is the full directory path to the root of your website where ToplistX is installed. For example, when you access http://www.yoursite.com/index.html the index.html file is located in your Document Root folder. ToplistX will try to automatically set this value for you, so in most cases you do not need to change this setting.

    ToplistX URL - This is the the base URL where ToplistX is installed on your server

    Banner URL - This is the URL that points to the directory on your server where you want member banners stored if you choose the option to download and serve them from your server. This should point to an empty directory with 777 permissions that is located inside the Document Root.

    Cookie Domain - The domain name to use when setting cookies. This must contain the domain name of the website where ToplistX is installed.

    Page Permissions - The file permissions to set on the generated ranking pages.

    E-mail Address - This is the e-mail address will appear in the From field of all e-mails sent by ToplistX

    E-mail Name - This is the name that will appear in the From field of all e-mails sent by ToplistX

    E-mail Sender - Choose the method that ToplistX will use for sending emails. In most cases you should leave this set to the default PHP mail() Function option.

      PHP mail() Function - ToplistX will use PHP's built in mail() function for sending e-mail messages.
      Sendmail - ToplistX will use sendmail on your server to send e-mail messages. This will require that PHP is not running in safe mode and the popen() function is not disabled. You will need to enter the full path to sendmail in the text input box after the drop down selection field.
      SMTP Server - ToplistX will use a SMTP server to send e-mail messages. This will require that the SMTP server has authorized your server to send e-mail messages through it. You will need to enter the hostname of the SMTP server in the input box after the drop down selection field.


    Date Format - The string that specifies the format to use for date values

    Time Format - The string that specifies the format to use for time values

    Your Timezone - Select the timezone you are located in so that dates and times can be calculated properly

    Decimal Point - The value to use for the decimal point in numbers

    Thousands Separator - The value to use for the thousands separator in numbers

    Using cron to automate page building, hourly stats, and daily stats - Check this box only if you have setup cron to handle the rebuilding of your pages and calculation of the hourly and daily stats.

    Rebuild Interval - The minimum amount of time, in seconds, between page rebuilds. This only has an affect if you are not using cron for rebuilds.
Ranking Images Settings The ranking images feature of the software allows you to display an image on your member sites that shows the current position of that site in the ranks. For example, if a member site is currently ranked #2 you can have an image display on their site that says they are ranked #2 in your list. More information can be found in the Ranking Images section of the software manual.
    Ranking Images URL - The URL where you will place the ranking images to display on your member sites

    Images To Show - The number of ranking images you will create with numbers on them

    File Extension - The file extension of the ranking images you have created.
Click Tracking & Rating Settings
    Tracking Mode - Choose the method that you want the software to use for statistics tracking. More information on the software stat tracking system can be found in the Statistical Tracking Overview section of the manual.

      Accounts send surfers directly to your site - With this method, account holders will be able to send surfers to any URL on your site that you specify, as long as you are able to include the in.php click tracking script as part of that page (using SSI, PHP, or an <img> tag). This is generally a better choice for search engine optimization.

      Unique link for each account - With this method, each member account will be given a unique link that they will need to place on their sites to track the surfers that they send to your site. The link will point to the ToplistX in.php script and contain their account username in the URL for tracking purposes.

    Secret Key - A unique value used with the Accounts send surfers directly to your site tracking mode. This can be any string of characters, but may only contain English letters, numbers, and the underscore character. You will need to use this value in your embedded tracking code, as described in the Statistical Tracking Overview section of the manual. DO NOT SHARE THIS VALUE WITH ANYONE!

    Tracking URL - The URL that member accounts will use to track their statistics. If you are using Unique link for each account, this must point to the ToplistX in.php script. If you are using the Accounts send surfers directly to your site, this must point to the page on your site that has the in.php script embedded.

    Default Forward URL - If you are using Unique link for each account, this is the URL that surfers will be sent to after their click is recorded by the in.php script.

    Alternate Out URL - If you are using the skimming feature of the out.php script, this is the URL that surfers will be sent to when none of your accounts are owed clicks but the surfer should be sent to one of your accounts based on the skim percentage.

    Redirect Status Code - The HTTP status code that the software should use when redirecting surfers to your member sites. In most cases you will want to use 301 for this value, although you could select any of the 30x HTTP status codes.

    Maximum Site Rating - The maximum value a surfer can select for a site rating.

    Comment Length - The minimum and maximum number of characters allowed in a surfer comment.

    Comment Interval - The amount of time, in seconds, that a user must wait before submitting another comment for the same account.

    Review new surfer comments before displaying them on your site - Check this box if you want to review surfer comments before they appear on your site
Account Submission Settings
    Account Submissions

      Open - You are currently accepting new accounts
      Closed - You are not currently accepting new accounts; this will disable the account submission form


    Review new accounts before listing them - Check this if you want to review all newly submitted accounts before they are listed on your site

    Review account editing before listing the changes - Check this if you want to review all changes made to accounts by the account holders

    New accounts must be confirmed through the submitter's e-mail address - Check this if you want all new account holders to confirm their account submission by e-mail

    Send confirmation e-mail message to new accounts when created - Check this if you want an e-mail message sent to new account signups when their account is added

    Site Title Length - The minimum and maximum number of characters allowed in an account site title

    Description Length - The minimum and maximum number of characters allowed in an account description

    Allow user to submit keywords with their site information - Check this if you want to allow keywords to be included with account signups

    Keywords Allowed - The maximum number of keywords that can be submitted with an account

    Default Return Percent - The default return percentage assigned to newly submitted accounts

    Maximum Banner Width - The maximum banner width, in pixels, allowed for submitted banners

    Maximum Banner Height - The maximum banner height, in pixels, allowed for submitted banners

    Maximum Banner Filesize - The maximum banner filesize, in bytes, allowed for submitted banners

    Force all banners to the height and width entered above - Check this if you want to assign the same width and height to all banners, no matter what their actual size is

    Download banners to check height, width, and filesize - Check this to have the software download and check the actual size of the banner

    Host member account banners from your server - Check this if you want to serve all account banners from your server

    Allow redirecting site URLs to be submitted (300 level HTTP status codes) - Check this if you want to allow accounts submitted that redirect surfers
Verification Code Settings

Please also review the Verification Codes section of this manual
    Font Directory - The full directory path to the location of the ToplistX fonts directory.

    Code Length - The minimum and maximum number of characters that will appear in a verification code.

    Use words file for verification codes - Check this box to use the values stored in the words file for the verification codes.

    Require verification code on account creation interface - Check this box to require a verification code during account submission.

    Require verification code on account rating interface - Check this box to require a verification code for surfers rating your member sites.

    Require verification code on account comment submission interface - Check this box to require a verification code for surfers leaving comments on your member sites.
Confirmation Message Settings
    Uncheck any of these boxes if you do not want a confirmation message to appear in the control panel interface when you execute the associated function.
